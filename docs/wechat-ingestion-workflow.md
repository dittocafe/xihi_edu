# 微信文章收录与改写流程

这个流程用于把公众号文章、朋友圈转发、家长群资料等信息，转成 Xihi 留学伴读手记可以长期调用的资料。核心原则是：先收录事实和问题，再形成自己的判断框架，不直接搬运原文。

## 1. 收录层：来源卡片

每看到一篇值得保存的微信文章，先建一张来源卡片，放在本地 Obsidian 或 `work/xihi-vault/10_sources/`。

来源卡片只回答几个问题：

- 这篇文章来自哪里，什么时候看到的。
- 它对哪个阶段、哪类家庭、哪个决策问题有用。
- 原文里哪些是事实、经验、观点、广告或引流话术。
- 它能支持我们未来写哪篇文章，或者补充哪篇文章。
- 是否需要二次核验，比如官方政策、学校官网、考试机构官网。

## 2. 判断层：信息卡片

一篇公众号文章不要马上改写成公开文章。先转成一张信息卡：

- 家长真正的问题是什么。
- 这篇文章给出的有效信息是什么。
- 哪些内容可能有机构立场。
- 北京、上海、外地家庭分别应该如何理解。
- 这条信息应该进入哪个路径页：高中阶段、本科阶段、陆本申研、资源库。

## 3. 发布层：原创文章

公开网站只发布自己的文章。文章可以参考多个来源，但表达方式、结构和结论都应该是 Xihi 自己的。

公开文章建议包含：

- 一个明确的家长决策问题。
- 一套判断框架，而不是简单推荐。
- 来源如何影响判断的简短说明。
- 不复制原文段落，不保留公众号广告话术。
- 需要时在 frontmatter 的 `source_refs` 字段记录来源名和用途。

## 4. 最小可行操作

第一阶段不用自动化。你只需要把微信文章链接或截图发给我，我按这个顺序处理：

1. 建来源卡片。
2. 打标签：阶段、路径、目的地、决策主题、资源类型。
3. 判断是否值得写成公开文章。
4. 如果值得，产出一篇网站文章草稿。
5. 文章进入本地预览，确认后再发布到 GitHub Pages。

## 5. 版权边界

不把公众号全文同步到公开仓库，不直接改写单篇文章发布成“近似文章”。更稳的方式是多来源交叉整理，输出自己的判断框架，并保留来源线索供内部追溯。
