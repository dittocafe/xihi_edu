# 公众号自动跟踪可行性评估

## 结论

公众号自动跟踪可以做，但第一版不建议做成“全自动抓取全文”。更合适的是半自动：

- 人工选择公众号和文章入口。
- 工具记录文章标题、链接、发布时间、公众号名称、摘要和标签。
- 需要分析时再人工确认或交给 AI 生成来源卡片。

这样更稳定，也更不容易踩到平台限制、登录风控和版权问题。

## 信息源分层

1. 机构公众号
   用来观察时间节点、活动包装、常见销售话术和市场趋势。价值高，但立场明显。

2. 官方信息源
   包括学校官网、College Board、Common App、签证、考试机构、大学项目页。适合做最终核验。

3. 家长经验源
   包括家长群、社群分享、朋友圈转发、知乎/小红书/论坛经验帖。适合发现问题，但要谨慎归纳。

4. 数据和榜单源
   包括录取数据、学校数据、专业就业数据、排名。适合做背景，但不能替代家庭决策。

5. 时间节点源
   AP、夏校、早申、常规申请、签证、入学、选课、找实习等节点提醒。适合放首页更新和季节性文章。

## 自动化等级

### Level 0：手动收录

你把链接、截图、复制文本或文章标题发给我，我生成来源卡片和文章草稿。现在就可以跑。

### Level 1：半自动清单

建立一个固定公众号名单。每周人工打开一次，把值得看的文章链接放进收录箱。工具负责整理、去重、打标签、生成候选主题。

### Level 2：浏览器辅助

在本机浏览器已登录微信/搜狗等入口的情况下，用浏览器自动打开、检索、保存标题和链接。这个方式可行，但会受登录状态、验证码、页面变化影响。

### Level 3：全自动抓取

长期无人值守抓取公众号文章全文。技术上可能有办法，但稳定性、合规性和维护成本都不理想，当前不建议作为主路线。

## 第一批建议跟踪字段

- source_name：公众号名，例如“谢强”
- source_account：微信号，例如“dalianxieqiang”
- source_type：机构公众号、个人顾问、官方信息、家长经验、数据来源
- original_title：原文标题
- original_url：原文链接
- published_at：原文发布时间
- captured_at：收录时间
- pathway：高中阶段、本科阶段、陆本申研、资源库
- stages：9年级、10年级、11年级、12年级、大一到大四
- decision_topics：顾问选择、夏校、科研、标化、选校、文书、就业、申研等
- useful_points：有价值的信息点
- bias_or_ad：可能的广告或立场
- verification_needed：是否需要官方核验
- next_action：存档、补充旧文、写新文、忽略

## 推荐第一版

先做 Level 0 + Level 1。也就是：固定公众号名单和来源卡片模板先跑起来，每周整理一次。等 20-30 篇来源卡片之后，再判断哪些环节值得自动化。
